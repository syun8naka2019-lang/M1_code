JavaScript Security Configuration Example

Content Security Policy (CSP) Configuration

This page demonstrates secure CSP headers for JavaScript applications.

<meta http-equiv="Content-Security-Policy" 
      content="default-src 'self'; 
               script-src 'self' 'unsafe-inline' https://cdn.jsdelivr.net; 
               style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; 
               font-src 'self' https://fonts.gstatic.com; 
               img-src 'self' data: https:; 
               connect-src 'self' https://api.example.com; 
               frame-src 'none'; 
               object-src 'none'; 
               base-uri 'self'; 
               form-action 'self'; 
               upgrade-insecure-requests;">

Security Features Enabled:

  • X-Content-Type-Options: nosniff - Prevents MIME-type sniffing
  • X-Frame-Options: DENY - Prevents clickjacking attacks
  • X-XSS-Protection: 1; mode=block - Enables XSS protection
  • Referrer-Policy: strict-origin-when-cross-origin - Controls referrer information

Secure JavaScript Practices

This section demonstrates secure JavaScript coding practices.

Environment Variable Usage

API endpoints and sensitive data should be loaded from environment variables:

// Secure API call example
const API_URL = process.env.API_URL || 'https://api.example.com';
const API_KEY = process.env.API_KEY;

async function secureApiCall() {
    try {
        const response = await fetch(`${API_URL}/data`, {
            method: 'GET',
            headers: {
                'Authorization': `Bearer ${API_KEY}`,
                'Content-Type': 'application/json'
            }
        });
        return await response.json();
    } catch (error) {
        console.error('API call failed:', error);
        throw error;
    }
}
